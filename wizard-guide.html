🔮 Complete Wizard Guide

Master every feature of the 4-step attribute creation process.

Step 1: Data Info

Configure the foundational settings for your attribute system:

🏷️ Basic Information

  • Data Name: Display name for your collection
  • Description: Purpose and context

⚙️ Code Generation

  • Namespace: C# namespace (e.g., "Game.Attributes")
  • Class Prefix: Prefix for classes (e.g., "Character")
Generated Classes Preview: ├── CharacterAttributeEnum.cs (type-safe IDs) ├── CharacterAttributes.cs (runtime container) ├── CharacterAttributeData.cs (ScriptableObject) └── CharacterUIBinding.cs (UI integration)
⚠️ Naming Rules:
  • Use PascalCase for class prefixes
  • Avoid Unity reserved names (Component, GameObject, etc.)
  • Keep namespaces short and logical

Step 2: Attribute Configuration

Build your attributes using the enhanced property drawer:

🏷️ Identity & Presentation

  • Name: Full attribute name (e.g., "Health Points")
  • Code: Short abbreviation (HP, MP, STR) - auto-generated
  • Icon: Sprite for UI display
  • Description: Tooltip and documentation text

⚙️ Behavior Types

🔷 Basic

Core stats like Strength, Dexterity

  • No special restrictions
  • Direct value storage
  • Perfect for primary stats

❤️ Vital

Health/Mana resources

  • Current/Max value system
  • Can regenerate over time
  • Event system (OnReachedZero, etc.)

📊 Percentage

Rates and chances

  • Critical rates, dodge chance
  • No value limits
  • Can exceed 100%

💰 Resource

Gold/XP/Level

  • Automatically integer-only
  • Positive minimums enforced
  • Perfect for currencies

🔄 Regeneration System (Vital Only)

  • Can Regenerate: Enable automatic recovery
  • Regeneration Rate: Points per second
  • Regeneration Delay: Cooldown after damage
💡 Smart Features:
  • Auto-generate codes from names with the "Auto" button
  • Resource types automatically lock Force Integer
  • Regeneration fields only appear for Vital types
  • Real-time validation prevents invalid configurations

Step 3: Code Generation Settings

All files are generated automatically - no configuration needed!

✅ Attribute Enum

Type-safe attribute IDs for runtime access

✅ Attribute Class

MonoBehaviour container with caching and validation

✅ AttributeData ScriptableObject

Data asset for tools and serialization

✅ UI Binding Class

Automatic UI synchronization system

📁 File Locations:
  • Generated files save to: Assets/Generated/
  • Namespace folders created automatically
  • Conflict detection shows overwrite warnings
